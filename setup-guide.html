📋 系統架構

這套系統做什麼，由哪幾塊組成

每個美股交易日收盤後，GitHub Actions 自動跑：

  1. 讀 accounts.json 列出的所有帳戶
  2. 每個帳戶依其指派的 策略 JSON（如 top10 / d2p2t6 / mom_6m_t20）計算當日應有持股
  3. 對比 Alpaca 當前持倉，下「最小量換股」訂單
  4. 生成 dashboard JSON 並推到 GitHub Pages
  5. 透過 SendGrid 寄每日報告到你的 email

三個 repo（公開 + 私有混合）

Repo用途可見性
tech-rebalance（主 repo） 所有 Python 程式碼 + Actions workflow + 策略 JSON + 持倉狀態 建議 Private（含交易紀錄）
tech-rebalance-dashboard GitHub Pages 入口，顯示績效、回測、log 必須 Public（免費 plan Pages 要求）
💡 為何兩個 repo？ GitHub 免費 plan 不支援 Private repo 的 Pages。我們把 Python 程式（含 API key 痕跡）放 Private，把純 HTML/JSON dashboard 放 Public，兩者透過 Actions 同步。

②申請 Alpaca 帳號

⏱️ 20 分鐘（含身分驗證）· 取得 API Key 與 Secret
  1. 到 https://alpaca.markets → Sign Up
  2. 填基本資料 → 上傳身分證件（線上拍照）→ 等審核（通常 1-2 小時）
  3. 登入 Dashboard → 左側選單 Paper Trading（不是 Live）
  4. 右上角 View API Keys → Generate New Key
  5. 記下 API Key ID 與 Secret Key（Secret 只顯示一次！）

每個策略要不同 Alpaca 帳戶嗎？

Paper Trading 一個帳戶就夠（虛擬資金 $100K）。實單要分開：每個策略獨立帳戶才能追蹤各自績效。Alpaca 一個 email 可以開多個 sub-account。

✅ 建議初次設定 先用 Paper Trading 1 個帳戶，跑通流程 + 看 1-2 個月績效，再考慮開實單帳戶。
⚠️ Secret Key 安全 永遠不要把 Secret 貼到 chat / Slack / 程式碼裡。只能放 GitHub Secrets。

③設定郵件寄送

⏱️ 10-15 分鐘 · 二選一：SMTP（推薦免費）或 SendGrid（也免費）

系統自動偵測：若有 SMTP_HOST 環境變數則走 SMTP，否則走 SendGrid。 也可用 EMAIL_PROVIDER=smtp|sendgrid 明確指定。

選項 A：SMTP（推薦，零第三方依賴）

用你既有 Gmail / Outlook / Yahoo 帳號。無需註冊新服務。

Gmail SMTP 設定

  1. 啟用 兩步驟驗證：Google 帳戶 → 安全性
  2. 產生 App Password：App passwords → 選 "Mail" + "Other (Custom name)" 輸入 "tech-rebalance" → 產生
  3. 記下 16 字元密碼（如 abcd efgh ijkl mnop，含空格也可，貼到 Secret 時系統會自動處理）

Outlook / Hotmail SMTP

  1. 啟用兩步驟驗證後到 App passwords 產生
  2. SMTP_HOST 用 smtp-mail.outlook.com，port 587

Yahoo SMTP

  1. 到 Account Security → Generate app password
  2. SMTP_HOST 用 smtp.mail.yahoo.com，port 465 SSL 或 587 TLS

各供應商對照

供應商SMTP_HOSTSMTP_PORT免費額度
Gmailsmtp.gmail.com587 (TLS) or 465 (SSL)500/天
Outlooksmtp-mail.outlook.com587 (TLS)300/天
Yahoosmtp.mail.yahoo.com465 (SSL) or 587 (TLS)500/天
iCloudsmtp.mail.me.com587 (TLS)1000/天
自架（Postfix 等）你的 server視設定無限
📋 SMTP 只需 1 個 Secret：SMTP_URL

已知 domain 的 email（gmail/outlook/yahoo/icloud …），系統從 EMAIL_RECIPIENT 自動推 provider。 SMTP_URL 直接填 App Password 即可。 

# 最短：純密碼（recipient 是 @gmail.com → 自動用 Gmail）
abcdEFGHijklMNOP

# 若 domain 不在已知清單，加 provider 短碼
gmail:abcdEFGHijklMNOP
outlook:YOUR_PASSWORD
yahoo:YOUR_PASSWORD

# 若 SMTP 登入帳號 ≠ 收件人
gmail:other@gmail.com:abcdEFGHijklMNOP

# 自架 server / 不在清單 → 完整 URI
smtp://user:pass@your-server.com:587
smtps://user:pass@your-server.com:465

自動 domain 對應： gmail.com、googlemail.com → gmail； outlook.com、hotmail.com、live.com、msn.com → outlook； yahoo.com、yahoo.com.tw、ymail.com → yahoo； icloud.com、me.com、mac.com → icloud。

provider 短碼映射的 SMTP serverport模式
gmailsmtp.gmail.com587TLS
outlook / hotmailsmtp-mail.outlook.com587TLS
yahoosmtp.mail.yahoo.com465SSL
icloudsmtp.mail.me.com587TLS
office365smtp.office365.com587TLS
💡 簡化原則 Gmail / Outlook / Yahoo / iCloud 用戶 不需要記 host 跟 port， 只要寫 provider:email:password 即可。系統知道 smtp.gmail.com / 587 / TLS 這些細節。

選項 B：SendGrid（HTTP API，更穩定）

SendGrid 免費 plan 每天 100 封。優點：純 HTTP，不擔心 SMTP 連線問題、IP 黑名單。

  1. 到 https://sendgrid.com → Start for Free
  2. 填資料註冊（不需信用卡）
  3. 登入後做 Sender Authentication（驗證寄件人 email）：
    • Settings → Sender Authentication → Verify a Single Sender
    • 填你的 Gmail（這會成為寄件人）→ 點確認信
  4. 產生 API Key：
    • Settings → API Keys → Create API Key
    • 選 Full Access
    • 記下 SG.xxxxxxxx（只顯示一次）

穩定性良好（偶爾 timeout）優秀

適合個人 1-3 帳戶多帳戶或商用

④設定 GitHub Secrets

⏱️ 10 分鐘 · 把 API keys 安全地放進 GitHub

到你的 repo 設定頁：github.com/YOUR_USERNAME/tech-rebalance/settings/secrets/actions

必填 Secrets

Secret 名稱值來自說明
ACC1_ALPACA_KEY 步驟 ② Alpaca 第一個帳戶的 API Key ID
ACC1_ALPACA_SECRET 步驟 ② Alpaca 第一個帳戶的 Secret Key
ACC1_BASE_URL 固定值 https://paper-api.alpaca.markets（Paper）
實單改 https://api.alpaca.markets
EMAIL_RECIPIENT 你想收信的 email（同時自動當寄件人） e.g. itemhsu@gmail.com
EMAIL_SENDER（選填） 若寄件人想跟收件人不同才設 未設 → 自動 = EMAIL_RECIPIENT
DASHBOARD_PUSH_TOKEN GitHub PAT 讓主 repo workflow 能 push 到 dashboard repo（下方說明）

選一組郵件 Secrets（A 或 B）

選項 A：SMTP（推薦，只需 1 個 Secret）

Secret 名稱範例值
SMTP_URL abcdEFGHijklMNOP（純密碼）

當 EMAIL_RECIPIENT 是已知 domain（gmail/outlook/yahoo/icloud …），SMTP_URL 直接填密碼。
自動推導：domain → provider → host/port/SSL/登入帳號全部自動。
未知 domain 才需 provider:password 或完整 smtp://... URI。

選項 B：SendGrid

Secret 名稱值來自
SENDGRID_API_KEY步驟 ③ SendGrid，SG.xxxx...
系統如何選擇 provider？
  • 有 SMTP_URL → 走 SMTP
  • 沒 SMTP_URL，有 SENDGRID_API_KEY → 走 SendGrid
  • 都沒設 → 不寄，僅 log
  • 明確指定可設 EMAIL_PROVIDER=smtp 或 =sendgrid

多帳戶？再加 ACC2 / ACC3 ...

每多一個帳戶就加三個 Secret：ACC2_ALPACA_KEY、ACC2_ALPACA_SECRET、ACC2_BASE_URL。

產生 DASHBOARD_PUSH_TOKEN（PAT）

  1. 到 GitHub → Settings → Developer settings → Fine-grained PAT
  2. Generate new token
  3. Repository access: Only select repositories → 選 tech-rebalance-dashboard
  4. Permissions: Contents → Read and write
  5. 產生後複製整段 github_pat_xxxx 到 DASHBOARD_PUSH_TOKEN
用 gh CLI 一次設好（SMTP + Gmail 範例） 
# Alpaca + 收信 email + dashboard
gh secret set ACC1_ALPACA_KEY    --body "你的key"
gh secret set ACC1_ALPACA_SECRET --body "你的secret"
gh secret set ACC1_BASE_URL      --body "https://paper-api.alpaca.markets"
gh secret set EMAIL_RECIPIENT    --body "you@gmail.com"     # 唯一要填的 email
gh secret set DASHBOARD_PUSH_TOKEN --body "github_pat_xxx"

# 選項 A：SMTP（純密碼，provider 自動從 EMAIL_RECIPIENT domain 推）
gh secret set SMTP_URL --body "abcdEFGHijklMNOP"

# 選項 B：SendGrid（A、B 二選一即可）
gh secret set SENDGRID_API_KEY --body "SG.xxx..."

✅ 驗證：先跑 DRY-RUN 確保流程通

🔧 新增 / 修改策略

⏱️ 5-10 分鐘 · 純 JSON 設定，零 Python 修改

系統用 v3 Schema 描述策略，加新策略只要寫一個 JSON 檔。

範例：新建「科技 50 + 9m 動能」策略

# 1. 複製模板
cp strategies/mom_6m_t20.json strategies/mom_9m_t50.json

# 2. 編輯關鍵欄位：
{
  "id": "mom_9m_t50",
  "meta": {"name": "MOM-9m 科技 50 動能"},
  "selection": {
    "method": "weighted_percentile",
    "n": 10, "watchlist_top_n": 50, "watchlist_metric": "market_cap"
  },
  "ranking": {
    "factors": [
      { "field": "price_momentum_9m", "weight": 1.0, "direction": "desc" }
    ]
  },
  "backtest": {
    "color": "#a855f7", "group": "momentum", "order": 110
  }
}

# 3. 在 accounts.json 加帳戶或修改既有帳戶
{"id": "3", "strategy": "mom_9m_t50", ...}

# 4. push
git add strategies/mom_9m_t50.json accounts.json
git commit -m "feat: add mom_9m_t50 strategy"
git push

排程下次跑時自動採用新策略。回測也會自動納入 momentum dashboard。

可用 selection methods

top_n_by_metric依單一指標排序取前 N（如純市值）
top_n_per_group分組各取 N 檔（如 D2P2T6）
factor_score多因子加權打分
weighted_percentilewatchlist 預篩 + 單因子（典型動能）
weighted_percentile_per_group分組 watchlist + 因子排序
buy_and_hold買入持有不調整（基準用）

👀 監控與除錯

怎麼確認系統健康、出錯怎麼查

每日例行檢查

  • 早上 8 點檢查 inbox：應有「每日報告」email
  • 沒收到 → 到 GitHub Actions 看排程 run 是否成功
  • NAV 變動異常 → 看 email 內的「今日交易」段
  • 有交易 → log 頁面看詳細事件（trigger / reason / 守則）

常用除錯指令

# 看最近 5 次 daily 執行狀態
gh run list --workflow=daily_all_accounts.yml --limit 5

# 看失敗 run 的 log（找特定 step）
gh run view RUN_ID --log-failed

# 手動觸發補跑（dry-run 安全測試）
gh workflow run daily_all_accounts.yml -f dry_run=true

# 看本機 trade_events（jsonl 格式可 jq 查詢）
jq 'select(.type=="ORDER_PLANNED")' data/1/trade_events.jsonl

常見錯誤

症狀原因修法
workflow 0 秒失敗 YAML 語法錯（多半是 PR merge 衝突） 用 python3 -c "import yaml; yaml.safe_load(open('.github/workflows/foo.yml'))" 驗
Alpaca 金鑰未設定 Secret 名稱與 accounts.json 的 prefix 對不上 確認 ACC{N}_ALPACA_KEY 對應 "alpaca_secret_prefix": "ACC{N}"
SendGrid 401 API key 過期或寄件人未驗證 SendGrid → Sender Authentication 重新驗證
SMTP 535 驗證失敗 用了登入密碼而非 App Password；或 2FA 未開 Gmail/Yahoo 必須開 2FA 後到 App passwords 產生 16 字元密碼
SMTP timeout GitHub Actions IP 被供應商 throttle，或 port 被擋 試切換 port：465 ↔ 587；或改用 SendGrid（HTTP 不會被 port 擋）
策略 schema validation 失敗 JSON 不符 v3 schema python3 -c "import json,jsonschema; ..." 在本機驗
Dashboard 顯示 404 Pages 未啟用 / push token 失效 步驟 ⑥ + 步驟 ④ DASHBOARD_PUSH_TOKEN
