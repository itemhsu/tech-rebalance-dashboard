🔴 Live（真錢）切換清單

paper 驗證通過後，把兩 repo 架構切到真錢帳戶 · 2026-06-04

真錢：下單由你親手不用 force，照月度排程paper 不停、可回退
最高原則：真錢下單由你本人在 GitHub 觸發/啟用排程，我不代按。金鑰只進 GitHub Secret 輸入框。 本清單只給步驟與檢查項，不含任何金鑰。

關卡 0 — 先過 paper（必須）

在做任何 live 設定前，先確認 paper 那筆真的成交且正確：
  • Alpaca paper → Orders：SELL V × 35、BUY ASML × 5 狀態為 filled。
  • Positions：V 歸零、ASML 出現、其餘 8 檔不變。
  • 沒有非預期的多筆/重複單。
✅ 都對 → 才往下。任何不對 → 停，先查清楚。

1. 觀念：live 跟 paper 差在哪

項目paperlive（真錢）
accounts.json 的 environment"paper"改成 "live"
連線網址（引擎自動切）paper-api.alpaca.marketsapi.alpaca.markets
ACC1_ALPACA_KEY/SECRETpaper 金鑰換成 live 帳戶金鑰
force 強制再平衡驗證時用過不要用 — 讓它照月度排程
排程關閉（手動）開啟（每月第一交易日自動）
引擎只看 environment 欄位決定打哪個網址；改 "live" + 換 live 金鑰，就會對真錢帳戶下單。

2. 建議：開「獨立的 live repo」（不要直接改 paper repo）

讓 paper 與 live 分開，paper 可繼續當對照、出事可回退。

  1. 用 examples/repo-b-template/（或複製 tech-rebalance-data）建一個 private repo，例如 tech-rebalance-live。
  2. 把 vendor/ 的引擎 wheel 一起帶過去（目前 v1.0.3）。
  3. 編輯 accounts.json：把 "environment": "paper" 改成 "live"。

3. 設 live 金鑰（你親手）

  1. Alpaca → 切到 Live（非 paper）→ 產生 live API Key / Secret。
  2. live repo → Settings → Secrets and variables → Actions → New repository secret：
    Name值
    ACC1_ALPACA_KEYlive API Key
    ACC1_ALPACA_SECRETlive Secret
    EMAIL_*（選用，要收報告才設）
live 金鑰是真錢權限，務必確認貼的是 live 帳戶的（不是 paper）。值只進 Secret 框，不寫進任何檔案。

4. 先做一次 live dry-run（不下單，再確認一次）

  1. live repo → Actions → Run workflow → dry_run: true、force: false。
  2. 看 log：應顯示 環境：live、連到 api.alpaca.markets、讀到你 live 帳戶的真實 NAV/持倉、算出計劃（不下單）。
  3. 確認 NAV/持倉是你 live 帳戶的真實數字 → 金鑰、環境都對。

5. 啟用每月自動排程（正式上線）

top10 是月度策略，真錢就讓它在每月第一個交易日自然換股，不用 force。

  1. 編輯 live repo 的 .github/workflows/daily.yml，把 schedule 那兩行取消註解： 
      schedule:
    - cron: '15 21 * * 1-5'   # 盤後（台灣 05:15 隔日）
  2. 之後每個交易日它會跑；守門只在每月第一交易日真的換股，其餘日子空轉（正確行為）。
  3. 不要手動 force 真錢下單；要手動就先 dry-run 看計劃。

6. 風險控管（強烈建議）

  • 小額起步：live 帳戶先放一小部分資金，跑 1～2 個月度週期確認穩定，再加碼。
  • paper 不要停：留著當對照，隨時比對 live 與 paper 的選股是否一致。
  • 第一次月度換股當天盯一下：開盤後到 Alpaca live 看訂單是否如 dry-run 預期成交。
  • 訂單追蹤已修（v1.0.3 get_order）：成交/拒單/未結會被記錄，這個安全網對真錢很重要。

7. 回退方案（出事立即用）

  1. live repo → Actions → 該 workflow → 關閉（Disable workflow），立刻停止自動下單。
  2. 必要時到 Alpaca live 手動取消未成交單 / 平倉。
  3. paper repo 一直都在，隨時可回去純觀察。
我能幫的：幫你建好 live repo 骨架（檔案、wheel、environment 改 live）、檢查 dry-run log、比對 live 與 paper 選股是否一致。 真錢的下單觸發與啟用排程由你親手；驗證與除錯我陪你做。
