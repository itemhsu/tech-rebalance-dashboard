🖥 管理者控制台 本機桌面 App

一個跨平台桌面 GUI，集中管理交易系統的所有「人工設定」： 帳戶 CRUD、券商/策略指派、GitHub Secrets、cron 排程、email 收件人等。 不跑 HTTP server、純本機執行、直接操作 repo 與 gh。

📋 執行摘要

一頁看懂

痛點：目前所有管理動作都要手改檔案 + 記指令 — 改 accounts.json、gh secret set、編輯 workflow 的 cron、git commit/push。 容易打錯、漏步驟（例如忘了設對應的 Secret）、也難一眼看全狀態。

解法：一個本機桌面 App，把這些動作收進有驗證、有防呆的 GUI。 它不是新的後端服務 —— 沒有 server、沒有資料庫，直接讀寫 repo 內的 accounts.json / workflow YAML，並 shell out 到 gh 設 Secret。

⚙️ 約束與技術選型

「無 HTTP server + 跨平台 + 本機」如何收斂選擇

關鍵約束「不依賴 HTTP server」直接排除所有 web 方案 （Flask/FastAPI + 瀏覽器、localhost dashboard）。剩下要在「原生桌面 GUI」裡選。

方案無 HTTP?跨平台與本專案契合評語
Python + PySide6 (Qt)✅ 原生✅ Mac/Win/Linux⭐⭐⭐⭐⭐推薦。專案本身 Python，GUI 可直接 import accounts 驗證/broker spec，零跨語言成本；Qt 介面專業
Python + Tkinter✅ 原生✅（內建）⭐⭐⭐⭐零額外依賴（Python 自帶）。介面陽春，但最穩、打包最小 → 備案
Tauri (Rust+webview)⚠️ IPC 非 HTTP✅⭐⭐二進位小、UI 漂亮，但要寫 Rust + 前端，跨語言橋接本專案邏輯，過重
Electron⚠️ IPC 非 HTTP✅⭐⭐技術上不需 HTTP server，但體積大、又是 JS 生態，與 Python 後端要 IPC 橋接
Web + 瀏覽器❌ 需 server—✗違反「不依賴 HTTP server」，出局
選定：Python + PySide6（Qt for Python） 無 HTTP/socket、跨平台、與既有 Python 系統同生態（可直接重用 brokers/registry.py 驗證 broker、strategies/*.json 列策略、 engine/universe_loader.py 等）。打包用 PyInstaller 出三平台單檔。
備案：若要求「零安裝相依」，退回 Tkinter（Python 內建）。

🔒 安全模型（金鑰）

最關鍵：金鑰絕不落地
  • App 永不持久化任何 secret。設 Secret 時，密碼欄輸入 → 直接以 stdin 餵給 gh secret set NAME --body - → 進 GitHub Secrets，不寫 accounts.json、不寫任何本機檔、不寫 log。
  • Secrets 分頁只顯示「存在 / 不存在」（來自 gh secret list），從不顯示值。
  • 輸入框遮罩、剪貼簿不留存、記憶體用完即清。
  • 每個帳戶依 secret_prefix + broker 算出「應有哪些 Secret」，缺的標紅 → 防「新增帳戶卻忘了設金鑰」這類漏步驟。
  • git push 含金鑰風險防護：commit 前掃描 diff，若疑似 token 字串出現在被 commit 的檔案 → 擋下並警告。
界線：App 幫你「呼叫 gh 設 Secret」， 但 gh 的登入授權仍由你本人完成（gh auth login）。 App 不碰你的 GitHub 帳密，只用你已授權的 gh session。
